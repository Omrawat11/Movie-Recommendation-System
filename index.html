    Under the Hood

    How CineSense Thinks

    A transparent look at the 4-step content-based filtering pipeline that powers every recommendation.

    01

    Metadata Collection

    Extract genres, keywords, overview, cast & director from each of the 4,803 TMDB movie records.

    02

    Tag Engineering

    Lowercase, stem (NLTK PorterStemmer), and merge all fields into a single tags string per movie.

    03

    Vectorization

    CountVectorizer transforms the tags corpus into a sparse 4,803 × 5,000 term-frequency matrix.

    04

    Cosine Similarity

    Measure the angular distance between the selected movie's vector and all 4,802 others; return the top 5.

    Cosine Similarity Formula

    cos(θ) = A · B ‖A‖ ‖B‖

    When cos(θ) → 1, the movies are highly similar in their tag-vector space.

    What CineSense AI Does

    CineSense AI is a content-based movie recommendation system. It analyzes 4,803 TMDB movies by engineering a composite tags field from genres, keywords, overview, cast, and director. These tags are stemmed using NLTK's PorterStemmer, vectorized into a 5,000-dimension sparse matrix via CountVectorizer, and compared with Cosine Similarity to find the top 5 most similar films for any query.

    This is not a classifier — there is no training/test split, no accuracy or precision metrics. The PKL file stores a preprocessed similarity matrix, not a trained model.
